# 2026-04-17

## 決定事項

### PSC戦略から「前日Vol干上がり」フィルターを削除
pain_count: 0
カテゴリ: trading-logic

高騰後押し目（PSC）戦略のエントリー条件から「前日出来高干上がり（prevVolume < avgVolume25）」を削除。

#### 経緯

当初、日中スキャンで毎回判定されている前日Vol干上がり条件が「候補銘柄を絞るために前処理化できないか」という観点から調査開始。→ 結果、フィルター自体がPF・期待値・MaxDDのすべてでマイナス効果と判明したため削除。

#### 検証結果（24ヶ月バックテスト: 2024-04-01 〜 2026-04-16）

| トレード数 | 198 | 340 |

| PF | 1.81 | 2.75 |

#### 判断根拠

- combined最大DDが **10.9% → 6.7%** に改善（-39%）— ノイズ増加どころかリスク低減
- PSC単体のPFが **2.20 → 4.59** に倍増 — 「干上がり」は高品質シグナルを除外していた
- GU側は256→240でほぼ影響なし（GUのPFは4.21→4.36で維持）
- 単体WFで「be不安定化」の懸念はあったが、combinedではデフォルトbe=0.3のまま改善するので実害なし

#### 削除した条件の解釈

「押し目で閑散 → 再加速」というストーリーは日本株中小型（≤2,500円）ユニバースでは機能せず、急騰後も出来高が高止まりしたまま再ブレイクするパターンの方が圧倒的に多い、という市場特性が浮き彫りになった。

#### 実装

- `entry-conditions.ts`: 5条件 → 4条件に変更
- `psc-scanner.ts` / `post-surge-consolidation-monitor.ts`: `prevVolume` 取得ロジック削除
- `api.ts` / `watchlist.ts`: UI表示から前日Vol干上がりを除去
- バックテスト・WF・combined それぞれから一時比較フラグを削除
