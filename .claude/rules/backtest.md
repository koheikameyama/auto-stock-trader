### プロとしての教訓(運用ルール)

- **Calmar比 主導の判断**: 期待値単独では判断しない(機会コスト・複利・リスク調整)
- **KPI優先順位**: Calmar > PF > 期待値 > 勝率
- **遅行指標は GU/PSC で機能せず**: ローリング勝率、エクイティSMAなど時間遅れ指標の "スロットル" は全て却下済み
- **構造的制御は有効**: cooldownDays(銘柄リピート抑制)、VIX scale(レジーム別)、breadth band(過熱veto)
- **機械的分散リスク管理は逆効果**: セクター上限は勝ち筋を削るだけ、シグナル品質の高い戦略では不要
- **集中が起きる時は"便乗すべき時"**: 強いセクターに機械的制限をかけない
- **mean-reversion は2024-2026の日本株で構造的に効かない**: continuation相場優勢、大型株のRSI<30到達は通常"真の悪材料"で続落
- **磨き上げ検証の必須プロトコル**: 24ヶ月単発BTで現状値より優位な値が出ても、WF検証で「過半数の窓で安定して選ばれる」ことを確認するまで本番反映しない。WFで不安定な値は単発測定のノイズである可能性が高い
- **既存戦略の磨き上げROIは限定的**: 2026-05-02 に PSC trail / GU gap×vol / RSI mean-reversion を 3連続で検証し、いずれも WF で否決。Calmar 9.37 は既に十分最適化されており、これ以上の単一パラメータ調整による安定的な改善は困難。今後は「磨き上げ」より「monitoring強化」「劣化早期検知」に時間を投資すべき
- **戦略はシーズン性を持つ**: 2026-05-21 過去9年検証で「リターンは D期 (2-3ヶ月続く大強気) 一発依存」が完全確証。過去9年で D期相当は約3年に1回 (2020-04, 2024-01, 2025-07)。**残り期間 (60%以上) は "offseason"** として "ゼロ近辺で耐える" のが戦略の本質。D期突入を取り逃さない監視と offseason 補完戦略 (米株ETF, 大型株戦略) の準備に時間を投資する
- **補完戦略はレジーム別の効果検証が必須**: 単独BTで PF 1.39 でも、既存戦略との結合効果(同時稼働時の機会競合 / 補完性)を見ないと意味がない。米株ETF はレジーム別検証で **idle 帯 PF 1.92 / band 帯 PF 1.08 / overheat 帯 PF 0.31** と判明し、idle 帯フィルター追加で全体 PF 1.39→1.83 / Calmar 4.12→5.24 に劇的改善。**「既存戦略 OFF 時にだけ動かす」設計が補完戦略の正解**

## バックテストの基本方針

**combinedバックテストが主。個別バックテストは診断用。**

実際のトレードは breakout と gapup が同時に資金を奪い合い、ポジション枠を競合する状態で動く。

| バックテスト | 位置づけ | 使うタイミング |
|---|---|---|
| **combined** | **メイン** | 本番パラメータ判断・定期的な汎化性能確認 |
| 個別（breakout/gapup） | 診断用 | どちらが問題かを切り分けるときのみ |

## 判断KPIの優先順位

**Calmar比を主KPIとする（期待値単独で判断しない）。**

| 優先度 | KPI | 計算式 | 目標値 |
|---|---|---|---|
| **1 (主KPI)** | **Calmar比** | NetRet / MaxDD | **≥ 3.0** |
| 2 | PF（Profit Factor） | Gross Profit / Gross Loss | ≥ 1.3 |
| 3 | 期待値（per-trade） | (勝率 × 平均勝%) - (敗率 × 平均負%) | > 0 |
| 4 | RR比 | 平均勝% / 平均負% | ≥ 1.5 |
| 5 | 資本稼働率 | 平均同時ポジション評価額 / 総資産 | 15% 以上 |

### 期待値単独で判断してはいけない理由

per-trade 期待値は**必要条件だが十分条件ではない**:

1. **機会コスト無視**: 期待値+3%×10回/年 < 期待値+1%×100回/年（自動化なら後者が可能）
2. **複利効果無視**: 継続運用では頻度も重要
3. **リスク調整無視**: 同じ期待値でもMaxDDが違えば運用可能性が違う

### パラメータ選択基準

複数パターンが接戦の場合:

1. **Calmar比で選ぶ**（最優先）
2. Calmar比が同等なら **シンプルな方**（条件数が少ない方）を選ぶ（Occam's razor / 過学習リスク低減）
3. それでも決まらなければ **パラメータ安定性**（WF各窓で同じパラメータが選ばれるか）で選ぶ

- **「即却下ライン確認」の価値**: 事前確率が低くても、イベントスタディ1本 (数十分) で判定できるなら測ってから殺す。予想が外れるケースがある (本件が初)。

### 事例: GU/PSC「1日1件制限」撤廃 → 枠まで複数エントリー（2026-07-03, KOH-505）

**契機**: 本番 GU スキャンで 4812.T(¥2,718, サージ5.77x)・4722.T(¥1,683, 1.54x) の2件がヒットしたが、首位 4812.T が集中率上限（1単元が資金50%超）で棄却され、**次点 4722.T は評価すらされずノーエントリー**に終わった。原因は `gapup-monitor.ts` / `post-surge-consolidation-monitor.ts` の `triggers.slice(0, 1)`（1日1件制限）。首位が資金都合で弾かれても次候補にフォールバックしない。

**判明した本番↔BTの構造的乖離**:
- **本番**: `slice(0,1)` で1日1件 + entry-executor が集中率50%チェック
- **combined BT / gapup・PSC 単体BT**: どちらも `maxDailyEntries` 未設定＝**枠(GU3/PSC2)まで当日複数エントリー**。combined BT は `maxDailyEntries` を一切参照していない
- → combined BT の全成績（Calmar 47 等）は「枠まで複数」前提で出ており、**本番だけ1日1件に絞られていた**。旧コメント「1日1件で PF 2.45→2.73」は gapup **単体WF**の話で、資金無限の理想値。CLAUDE.md 鉄則「正式判断は combined」に照らすと単体WFで判断していたのが誤り

**検証（`--compare-daily-entries` を combined-run に追加、本番影響なしフラグ）**:

単体WF（複数 vs 1件、`--max-daily-entries=1`）:

| | GU複数 | GU1件 | PSC複数 | PSC1件 |
|---|---:|---:|---:|---:|
| OOS集計PF | 2.15 | 2.53 | 2.58 | 2.56 |
| OOS平均PF | 3.26 | 3.07 | 2.99 | 2.81 |
| **IS/OOS比** | **0.99** | 1.39 | **1.00** | 1.21 |
| 総トレード | 273 | 140 | 271 | 141 |

→ 単体では per-trade PF は1件がやや上だが、**複数の方が IS/OOS比 ~1.0 で過学習なく頑健**。トレード数は複数が倍。

combined（正式判断基準、¥500K、GU/PSC の1日上限をスイープ）:

| 設定 | PF | MaxDD | NetRet | **Calmar** |
|---|---:|---:|---:|---:|
| **複数（枠まで, BT既定）** 長期2024-03〜 | 3.92 | 6.0% | +283.7% | **20.31** |
| GU=1/PSC=1（**旧本番**） 長期 | 3.06 | 10.6% | +178.0% | **7.16**（最下位） |
| GU=2/PSC=2 長期 | 3.85 | 6.6% | +292.7% | 19.07 |
| GU=複数/PSC=1 長期 | 3.98 | 6.6% | +289.5% | 18.69 |

- **正式判断基準の長期combined(28ヶ月)で、旧本番(GU=1/PSC=1)が全構成中ダントツ最悪**（Calmar 7.16, MaxDD 10.6%）。複数化(枠まで)で **Calmar 20.31 / MaxDD 6.0%** に激変
- 直近12ヶ月(D期偏重)の単発BTでは逆に「GU=1/PSC=複数」が最良に見えたが、**長期では複数(枠まで)が最良**。これは CLAUDE.md 鉄則「単発BTの優位はWFで確認するまでノイズ」の典型例。GU複数化が12ヶ月で悪く見えたのは D期特有のノイズだった

**判定: 1日1件制限は害。GU/PSC とも枠まで複数エントリーに統一（当初の狙い＝4722.T のような次点取りこぼし防止も自動達成）。**

**実装**（`gapup-monitor.ts` / `post-surge-consolidation-monitor.ts`）:
- `slice(0,1)` を撤廃し全トリガーをループ。トリガーは holdingTickers（保有＋当日pending買い）除外済み・優先度降順ソート済み
- 空き枠 = `MAX_POSITIONS_GU/PSC − 既存open/orderedポジ数` を算出、成功ごとに `slotsLeft--`、0で当日確定。同一ループ内の枠超過を防ぎ、二重建ては holdingTickers 除外で別途担保、executeEntry内の枠チェックも保険
- 失敗の分類: **枠上限/現金残高不足/予算不足/日次損失制限** = 当日打ち止め(break)、**集中率上限/投資比率上限/セクター/SLクランプ等の銘柄固有** = 次候補へ継続、**例外(catch)** = 一時障害で次分再試行。`result.retryable:true` は全て構造的発注不可、真の一時障害は例外側という分離を利用
- BT比較モード `--compare-daily-entries`（combined-simulation の `guMaxDailyEntries`/`pscMaxDailyEntries` + combined-run の比較ブロック）は再現用に残置（フラグ未設定で baseline 完全不変、commit 9eea86ab で先行マージ済み）

## RSI Oversold Bounce 戦略の検証と棄却（2026-05-02実施）

**結論: 棄却。mean-reversion 系は2024-2026の日本株（特に大型株）で構造的に機能しない。再検討禁止。**

A期（平穏ボックス相場）の弱点補完を目的に、大型株向けRSI mean-reversion戦略を実装してBTで検証。

### エントリー条件（検証時）

- RSI(14) < 30（売られすぎ）
- 直近5日リターン ≤ -5%（短期下落確認）
- 当日陽線（リバウンド開始）
- 200日SMA上（長期上昇トレンド中の押し目）
- ユニバース: 時価総額 ≥ ¥10B〜100B、価格 ≤ ¥100,000
- SL: ATR×1.5 or -7%、タイムストップ 5/7日
- 出口: SL/BE/トレール + RSI > 50で利確（mean-reversion完了）

### 構造的失敗の論理

2. **緩めるほど負ける**（NetRet -12.8% → -34.5%）= 埋もれたエッジを発掘したのではなく、ノイズに突っ込んでいる証拠

4. **WF検証は不要**（過学習以前の問題、IS/OOS両方で負ける）

### なぜ機能しないか（プロ分析）

- **2024-2026の日本株は continuation 相場が強い**（AI/半導体トレンド、大型株モメンタム優勢）
- **大型株のRSI<30到達は通常"真の悪材料"**（決算ミス、ガイダンス引き下げ、海外株安連鎖）→ 反転ではなく続落
- **「陽線で反転確認」は1日のノイズ**で、翌日続落が多い
- **A期の負けは別途 breadth band 55-80% 追加で既に解決済み**（NetPnL +¥13K黒字化、2026-04-21検証）

### 今後の禁止事項

以下のアイデアは本検証結果により**再検討禁止**:

- RSIベースの mean-reversion（RSI<30, RSI<35, RSI<40 全て負ける）
- 5日下落 + 陽線リバーサル系（緩めても改善しない）
- Bollinger Band下限tap → リバウンド系（同じ構造的失敗が予測される）
- 5日連続陰線後の陽線リバーサル系（同上）

mean-reversion 系を再検討する場合の最小限の前提条件:

- **市場環境が変化したことの実証**（例: 日経が trend → range に明確に転換した時期 6ヶ月以上）
- **別期間（2018-2022）のbackfillデータでpilot試験を先に実施**
- **A期相当のサンプルが20件以上集まる長期データ**で先に有意性を確認

それ以前にRSI/BB/連続陰線系のmean-reversion戦略を新規実装するのは時間の無駄。
