### プロとしての教訓(運用ルール)

- **Calmar比 主導の判断**: 期待値単独では判断しない(機会コスト・複利・リスク調整)
- **KPI優先順位**: Calmar > PF > 期待値 > 勝率
- **遅行指標は GU/PSC で機能せず**: ローリング勝率、エクイティSMAなど時間遅れ指標の "スロットル" は全て却下済み
- **構造的制御は有効**: cooldownDays(銘柄リピート抑制)、VIX scale(レジーム別)、breadth band(過熱veto)
- **機械的分散リスク管理は逆効果**: セクター上限は勝ち筋を削るだけ、シグナル品質の高い戦略では不要
- **集中が起きる時は"便乗すべき時"**: 強いセクターに機械的制限をかけない
- **mean-reversion は2024-2026の日本株で構造的に効かない**: continuation相場優勢、大型株のRSI<30到達は通常"真の悪材料"で続落
- **磨き上げ検証の必須プロトコル**: 24ヶ月単発BTで現状値より優位な値が出ても、WF検証で「過半数の窓で安定して選ばれる」ことを確認するまで本番反映しない。WFで不安定な値は単発測定のノイズである可能性が高い
- **既存戦略の磨き上げROIは限定的**: 2026-05-02 に PSC trail / GU gap×vol / RSI mean-reversion を 3連続で検証し、いずれも WF で否決。Calmar 9.37 は既に十分最適化されており、これ以上の単一パラメータ調整による安定的な改善は困難。今後は「磨き上げ」より「monitoring強化」「劣化早期検知」に時間を投資すべき
- **戦略はシーズン性を持つ**: 2026-05-21 過去9年検証で「リターンは D期 (2-3ヶ月続く大強気) 一発依存」が完全確証。過去9年で D期相当は約3年に1回 (2020-04, 2024-01, 2025-07)。**残り期間 (60%以上) は "offseason"** として "ゼロ近辺で耐える" のが戦略の本質。D期突入を取り逃さない監視と offseason 補完戦略 (米株ETF, 大型株戦略) の準備に時間を投資する

## バックテストの基本方針

**combinedバックテストが主。個別バックテストは診断用。**

実際のトレードは breakout と gapup が同時に資金を奪い合い、ポジション枠を競合する状態で動く。
個別バックテストは「資金が無限にある理想値」に過ぎないため、**正式な結果判断は combined で行う。**

| バックテスト | 位置づけ | 使うタイミング |
|---|---|---|
| **combined** | **メイン** | 本番パラメータ判断・定期的な汎化性能確認 |
| 個別（breakout/gapup） | 診断用 | どちらが問題かを切り分けるときのみ |

## 判断KPIの優先順位

**Calmar比を主KPIとする（期待値単独で判断しない）。**

| 優先度 | KPI | 計算式 | 目標値 |
|---|---|---|---|
| **1 (主KPI)** | **Calmar比** | NetRet / MaxDD | **≥ 3.0** |
| 2 | PF（Profit Factor） | Gross Profit / Gross Loss | ≥ 1.3 |
| 3 | 期待値（per-trade） | (勝率 × 平均勝%) - (敗率 × 平均負%) | > 0 |
| 4 | RR比 | 平均勝% / 平均負% | ≥ 1.5 |
| 5 | 資本稼働率 | 平均同時ポジション評価額 / 総資産 | 15% 以上 |

### 期待値単独で判断してはいけない理由

per-trade 期待値は**必要条件だが十分条件ではない**:

1. **機会コスト無視**: 期待値+3%×10回/年 < 期待値+1%×100回/年（自動化なら後者が可能）
2. **複利効果無視**: 継続運用では頻度も重要
3. **リスク調整無視**: 同じ期待値でもMaxDDが違えば運用可能性が違う

### パラメータ選択基準

複数パターンが接戦の場合:

1. **Calmar比で選ぶ**（最優先）
2. Calmar比が同等なら **シンプルな方**（条件数が少ない方）を選ぶ（Occam's razor / 過学習リスク低減）
3. それでも決まらなければ **パラメータ安定性**（WF各窓で同じパラメータが選ばれるか）で選ぶ

1. **allData の universe 分離**: `precomputeMomentumSignals` は top `topN*2` しか返さないため、combined で3,034銘柄の allData に対してランキングすると上位は小型株が独占 → 後filterで大型株ゼロ → 空集合になる。**precompute 前に `allDataForMom` (大型株のみ868銘柄)を分離して渡す必要がある**
2. **breadth計算の universe 問題**: combined の `precomputed.dailyBreadth` は全3,034銘柄基準。momentum標準BTは868銘柄基準。同じ閾値0.5でも発火日が異なる → MOMENTUM_LARGECAP_PARAMS で `marketTrendFilter: false` に固定し副作用回避
3. **lookback バッファ**: WF で IS startDate からの120日lookbackが必要 → `walk-forward-momentum.ts` で precompute に `isStartWithBuffer = isStart - (lookback+30)日` を渡す実装にしている

### 追記（2026-04-22）: 本番側にも VIX scale を適用

BT と本番の乖離を解消するため、`getRegimeRiskScale()` を共有ヘルパー化し、本番の `entry-executor.ts` のリスク計算にも適用:

- `src/core/market-regime.ts`: `getRegimeRiskScale(regime, custom?)` を export（BT/本番共通）
- `src/backtest/combined-simulation.ts`: local copy を削除して import に差し替え（挙動変化なし）
- `src/core/breakout/entry-executor.ts`: `todayAssessment.vix` から regime を判定、`streakAdjustedPct × regimeScale` で最終リスク%を算出

- 本変更により BT と本番の挙動が一致 = BT結果が信頼できる予測値になる

**w20 t45% s0.5 で MaxDD 7.9%(現状 9.0% から-12%)と最低を記録**。NetRet が-25%の代償。Calmar KPI では不採用だが、**MaxDD最小化を優先する運用スタイル**(例: 絶対リスク重視の個人投資家)では選択肢になり得る。

### 実装として残したもの（本番影響なし、運用スタイル変更時の選択肢）

- `src/backtest/combined-simulation.ts`: `SimContext.loseStreakScaling` オプション + `getStreakScale()` ヘルパー + 5戦略の quantity に regimeScale × streakScale 合成
- `src/backtest/combined-run.ts`: `--compare-streak` 比較モード

### 今後の方針

**Phase 0 + B3 の2回の検証で「遅行指標系スロットルは GU/PSC に効かない」が確定。**

- ローリング勝率・エクイティSMAなど時間遅れの指標はすべて却下
- 今後同種のアイデアが出た場合、本検証を根拠に即却下可能
- レジーム適応で改善したい場合は先行指標(breadth・VIX・Index SMA)を使うこと(既に実装済み)

## cooldownDays 調整(A3)検証（2026-04-22実施、現状最適を再確認）

**結論: 現状の cooldownDays=3 が最適。副次的発見として 0日運用では MaxDD 21.6% で壊滅することを確認、**
**cooldownは「構造的に有効な防御機構」であり B3/Phase 0 の遅行指標系スロットルとは別性質と判明。**

GU/PSC の cooldownDays=3 が WF で選ばれた値だったが、明示的な比較検証はしていなかった。
5日 / 10日 / 20日 / 0日(無し)を振って最適性を再確認。

- **MaxDD 10.0% → 21.6% (+116%)、Calmar 8.34 → 2.38 (-71%)** と劇的悪化

## RSI Oversold Bounce 戦略の検証と棄却（2026-05-02実施）

**結論: 棄却。mean-reversion 系は2024-2026の日本株（特に大型株）で構造的に機能しない。再検討禁止。**

A期（平穏ボックス相場）の弱点補完を目的に、大型株向けRSI mean-reversion戦略を実装してBTで検証。

### エントリー条件（検証時）

- RSI(14) < 30（売られすぎ）
- 直近5日リターン ≤ -5%（短期下落確認）
- 当日陽線（リバウンド開始）
- 200日SMA上（長期上昇トレンド中の押し目）
- ユニバース: 時価総額 ≥ ¥10B〜100B、価格 ≤ ¥100,000
- SL: ATR×1.5 or -7%、タイムストップ 5/7日
- 出口: SL/BE/トレール + RSI > 50で利確（mean-reversion完了）

### 検証結果（24ヶ月、2024-03 〜 2026-04、本番DB ¥500K）

| Variant | 条件 | tr | WR | PF | 期待値 | NetRet |
|---|---|---:|---:|---:|---:|---:|
| **A: 厳格** | RSI<30, -5%, SMA200, mcap≥100B | 23 | 17% | **0.13** | -2.40% | -12.8% |
| B: 緩和1 | RSI<35, -3%, SMA200, mcap≥10B | 86 | 27% | 0.20 | -1.51% | -28.0% |
| C: 緩和2 | RSI<35, -3%, SMA100, mcap≥10B | 105 | 28% | 0.21 | -1.54% | -31.1% |
| D: 緩和3 | RSI<40, -3%, SMA100, mcap≥10B | 140 | 31% | 0.25 | -1.29% | -34.5% |
| E: maxPos=3 | C と同条件 + 枠拡大 | 113 | 27% | 0.22 | -1.43% | -31.2% |

### 構造的失敗の論理

1. **全パターンで PF < 0.25**（1円勝つのに4-8円失う）= mean-reversionが成立していない
2. **緩めるほど負ける**（NetRet -12.8% → -34.5%）= 埋もれたエッジを発掘したのではなく、ノイズに突っ込んでいる証拠
3. **勝率17-31%**（mean-reversion標準55%+を大幅に下回る）
4. **WF検証は不要**（過学習以前の問題、IS/OOS両方で負ける）

### なぜ機能しないか（プロ分析）

- **2024-2026の日本株は continuation 相場が強い**（AI/半導体トレンド、大型株モメンタム優勢）
- **大型株のRSI<30到達は通常"真の悪材料"**（決算ミス、ガイダンス引き下げ、海外株安連鎖）→ 反転ではなく続落
- **「陽線で反転確認」は1日のノイズ**で、翌日続落が多い
- **A期の負けは別途 breadth band 55-80% 追加で既に解決済み**（NetPnL +¥13K黒字化、2026-04-21検証）

### 今後の禁止事項

以下のアイデアは本検証結果により**再検討禁止**:

- RSIベースの mean-reversion（RSI<30, RSI<35, RSI<40 全て負ける）
- 5日下落 + 陽線リバーサル系（緩めても改善しない）
- Bollinger Band下限tap → リバウンド系（同じ構造的失敗が予測される）
- 5日連続陰線後の陽線リバーサル系（同上）

mean-reversion 系を再検討する場合の最小限の前提条件:

- **市場環境が変化したことの実証**（例: 日経が trend → range に明確に転換した時期 6ヶ月以上）
- **別期間（2018-2022）のbackfillデータでpilot試験を先に実施**
- **A期相当のサンプルが20件以上集まる長期データ**で先に有意性を確認

それ以前にRSI/BB/連続陰線系のmean-reversion戦略を新規実装するのは時間の無駄。
