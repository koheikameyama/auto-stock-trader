# スコアリングシステム再設計

## 背景

現行の4カテゴリ100点満点スコアリングシステムは「スコアが高くても勝率が良くない」問題を抱えている。バックテスト結果（500銘柄×12ヶ月、157-168トレード）でPF 1.07と薄く、本番運用に必要なPF 1.3以上を達成できていない。

### 現行システムの構造的問題

1. **逆モメンタムRSI**: RSI 50-65にピークを置いているが、トレンドフォロー戦略ではRSI 60-70の銘柄の方がモメンタムが強い
2. **ファンダメンタルズの無関係性**: スイングトレード（3-10営業日保有）にPERやPBRは影響しない
3. **トレンド二重カウント**: MA配列とRS（相対強度）が似た情報を測っている
4. **タイミング信号の弱さ**: 「良い銘柄か」は測れるが「今入るべきか」が測れない
5. **流動性のスコア化**: 最低基準を満たさない銘柄は排除すべきで、点数を付ける意味がない

## 設計思想

**「この銘柄は良い銘柄か？」→「この銘柄に今入るべきか？」**

スイングトレードで重要なのは銘柄の品質ではなく、エントリータイミングの正確さ。トレンドが出ていて、押し目やブレイクアウトのタイミングで、リスクが管理可能な状態で入ることが勝率に直結する。

## アーキテクチャ

### 全体構造

```
ゲート（即死ルール） → 不合格なら即除外
  ↓ 合格
3カテゴリスコアリング（100点満点）
  ├─ トレンド品質    (40点)
  ├─ エントリータイミング (35点)
  └─ リスク品質      (25点)
```

### ゲート（即死ルール）

スコアではなくバイナリ判定。不合格なら候補から即除外する。

| ゲート | 条件 | 根拠 |
|--------|------|------|
| 流動性 | 25日平均出来高 >= 50,000株 | 約定リスク回避 |
| スプレッド | 株価 <= maxPrice設定 | 予算制約 |
| 最低ボラ | ATR(14)/終値 × 100 >= 1.5% | 低ボラ銘柄はスイングに不向き |
| 決算接近 | 決算発表まで5日超（カレンダー日数） | 決算ギャンブル回避 |
| 配当 | 権利落ち日まで3日超（カレンダー日数） | 権利落ち下落回避 |

**現行との違い**: 流動性（10点）をゲートに変格。ファンダメンタルズ（10点）を廃止。

**決算・配当の日数**: カレンダー日数で判定（現行と同じ）。営業日カレンダーは不要。
**配当データ欠損時**: `exDividendDate`がnullの場合、配当ゲートは合格（安全側デフォルト）。

## カテゴリ1: トレンド品質（40点）

「今トレンドが出ているか、そのトレンドは信頼できるか」を測る。

### 1-1. MA配列スコア（18点）

移動平均線のパーフェクトオーダーを評価する。

| 条件 | 点数 | 根拠 |
|------|------|------|
| 終値 > SMA5 > SMA25 > SMA75 | 18 | 完全パーフェクトオーダー |
| 終値 > SMA5 > SMA25、SMA75下 | 14 | 短中期は揃っている |
| 終値 > SMA25、SMA5割れ | 8 | 押し目の可能性 |
| SMA25上だが配列崩れ | 4 | トレンド弱い |
| SMA25下 | 0 | トレンドなし |

現行7段階を5段階に簡素化し、SMA75を加えて長期トレンドも考慮する。

### 1-2. 週足トレンド確認（12点）

日足だけでなく上位時間軸でトレンド方向を確認する。

| 条件 | 点数 |
|------|------|
| 週足SMA13上 & 週足SMA13が上向き | 12 |
| 週足SMA13上 & 横ばい | 8 |
| 週足SMA13下 & 上向き（転換初期） | 4 |
| 週足SMA13下 & 下向き | 0 |

日足だけで判断すると、週足下降トレンドのリバウンドに引っかかるリスクがある。週足SMA13（≒日足SMA65相当）でフィルタリングする。

**方向判定**: `上向き` = 今週のSMA13 > 前週のSMA13、`横ばい` = 変化率が±0.5%以内、`下向き` = 今週のSMA13 < 前週のSMA13。

### 1-3. トレンド継続性（10点）

トレンドの「鮮度」を測る。

| 条件 | 点数 | 根拠 |
|------|------|------|
| SMA25上に10-30日連続 | 10 | スイングのスイートスポット |
| SMA25上に5-9日 | 7 | トレンド初期 |
| SMA25上に31-50日 | 5 | やや成熟 |
| SMA25上に50日超 | 2 | 終盤リスク |
| SMA25下 | 0 | - |

トレンドは永遠に続かない。10-30日がスイングトレードで最も美味しい期間で、50日超は反転リスクが高い。

**カウント方法**: 最新のバーから逆順に、`close > SMA25`の連続日数をカウント。1日でも`close <= SMA25`があればカウント停止。休日は無視（営業日のみカウント）。

## カテゴリ2: エントリータイミング（35点）

「トレンドは良いとして、今がエントリーすべきタイミングか」を測る。現行システムで最も弱い部分。

### 2-1. プルバック深度（15点）

トレンド中の押し目の深さを評価する。

| 条件 | 点数 | 根拠 |
|------|------|------|
| SMA25付近（乖離率 -1%〜+2%）で反発サイン | 15 | 理想的な押し目買いポイント |
| SMA5-SMA25間（浅い押し目） | 10 | まだ浅いが入れる |
| SMA25を一時的に割った直後に復帰 | 8 | やや深いが反発確認 |
| SMA5上（押してない） | 3 | 飛び乗りリスク |
| SMA25大幅下（乖離-3%超） | 0 | トレンド崩壊の可能性 |

**乖離率の計算**: `deviationRate = (close - SMA25) / SMA25 * 100`

**反発サインの定義**:
```
reversal = false
for bar in [当日, 前日]:
    lowerShadow = min(open, close) - low
    realBody = abs(close - open)
    if lowerShadow >= realBody: reversal = true
if 前日が陰線(close < open) AND 当日が陽線(close > open): reversal = true
```

**SMA25一時割れ復帰（8点）**: 直近3営業日以内に`close < SMA25`の日が1日以上あり、かつ当日の`close > SMA25`。

**評価順序**: 条件は上から順に評価し、最初にマッチした条件のスコアを採用する。

SMA25への押し目はリスクリワードが最も良いエントリーポイント。「高いところで買わない」原則を数値化する。

### 2-2. ブレイクアウト検出（12点）

レンジからの放れを検出する。プルバックとは異なるエントリーパターン。

| 条件 | 点数 |
|------|------|
| 直近20日高値を出来高増加（当日/25日平均 > 1.5）で更新 | 12 |
| 直近20日高値を通常出来高で更新 | 7 |
| 直近10日高値更新（20日は未更新） | 4 |
| 高値更新なし | 0 |

プルバック深度とブレイクアウト検出は独立にスコアリングし加算する。条件の性質上、両方が同時に満点になることは稀だが、相互排他ルールは設けない。

**高値更新の判定**: `close > 直近20日間の最高値（当日除く）`を使用。ザラ場高値（high）ではなく終値ベース。

### 2-3. ローソク足シグナル（8点）

直近1-3本のローソク足パターンを補助的に評価する。

| 条件 | 点数 |
|------|------|
| 包み足（陽線）+ 出来高増加 | 8 |
| 長い下ヒゲ（実体の2倍超）+ SMA付近 | 6 |
| 連続陽線（3本）+ 出来高漸増 | 5 |
| 十字線 + サポート付近 | 3 |
| 特にシグナルなし | 0 |

ローソク足単体の信頼性は低いため8点に抑え、過大評価しない。複数パターンに該当する場合は最高スコアのパターン1つを採用する。

## カテゴリ3: リスク品質（25点）

「このトレードのリスクは管理可能か」を測る。エントリー後の損切り幅やボラティリティの安定性を評価する。

### 3-1. ATR安定性（10点）

ATR（平均真の値幅）の安定度を評価する。

| 条件 | 点数 | 根拠 |
|------|------|------|
| ATR14の変動係数(CV) < 0.15 | 10 | ボラが安定、損切り幅が読める |
| CV 0.15-0.25 | 7 | まあまあ安定 |
| CV 0.25-0.35 | 4 | やや不安定 |
| CV > 0.35 | 0 | 損切り幅が予測不能 |

変動係数(CV) = ATR14の直近20日間の標準偏差 / 平均値。ATRそのものではなく「ATRの安定性」を見る。ATRが日によって大きく変わる銘柄は、損切りラインの設定が難しく、想定外の損失が出やすい。

### 3-2. レンジ収縮度（8点）

ボリンジャーバンド幅でボラティリティ収縮（スクイーズ）を検出する。

| 条件 | 点数 |
|------|------|
| BB幅が直近60日で最小付近（下位20%） | 8 |
| BB幅が下位20-40% | 5 |
| BB幅が中央付近 | 3 |
| BB幅が上位40%（ボラ拡大中） | 0 |

収縮後にブレイクすると大きく動く傾向がある。ブレイクアウト検出との相性が良い。

**BB幅**: `BB上限(20,2σ) - BB下限(20,2σ)`を直近60営業日分計算し、当日のBB幅のパーセンタイル順位を求める。ローリング計算が必要なため、`calculateBBWidthHistory(ohlcv, period=20, lookback=60)` ヘルパーを新設する。

### 3-3. 出来高安定性（7点）

出来高の安定度と方向性を評価する。

| 条件 | 点数 |
|------|------|
| 出来高5日MA > 25日MA & CV < 0.5 | 7 | 安定的に出来高増加 |
| 出来高5日MA > 25日MA & CV 0.5-0.8 | 5 | 増加傾向だがムラあり |
| 出来高安定しているが減少傾向 | 3 | 関心低下 |
| 出来高が不安定（CV > 0.8） | 0 | 流動性リスク |

**出来高CV**: 直近25日間の日次出来高の標準偏差 / 平均値。

## スコア閾値

| ランク | スコア範囲 | アクション |
|--------|-----------|-----------|
| S | 80-100 | 最優先エントリー候補 |
| A | 65-79 | エントリー候補 |
| B | 50-64 | ウォッチリスト |
| C | 35-49 | 見送り |
| D | 0-34 | 対象外 |

閾値は実装後のバックテストで調整する。

## データ要件

### 必要なOHLCVデータ

| 指標 | 必要日数 | 根拠 |
|------|---------|------|
| SMA75 | 75日 | トレンド品質のMA配列 |
| 週足SMA13 | 65日（≒13週） | 週足トレンド確認 |
| BB幅の60日パーセンタイル | 80日 | レンジ収縮度 |
| ATR14のCV（20日） | 34日 | ATR安定性 |
| 出来高25日MA | 25日 | 出来高安定性 |
| 20日高値 | 20日 | ブレイクアウト検出 |

最低必要日数: 80日（BB幅計算用）。現行のon-the-flyモードのルックバック200日で十分カバーできる。

## 廃止項目

現行から以下を廃止する:

| 廃止項目 | 現行カテゴリ | 理由 |
|----------|------------|------|
| RSIスコア | テクニカル(65点内) | 逆モメンタム設計がトレンドフォローと矛盾 |
| MACDスコア | テクニカル(65点内) | MA配列と情報が重複 |
| RSスコア（相対強度） | テクニカル(65点内) | MA配列・週足トレンドと重複 |
| パターン認識 | パターン(15点) | ローソク足シグナル(8点)に再設計 |
| 流動性スコア | 流動性(10点) | ゲート（バイナリ）に変更 |
| ファンダメンタルズ | ファンダ(10点) | スイングトレードに無関係 |
| 逆張りウィナーボーナス | テクニカル(65点内) | 新設計に不適合 |
| 週足トレンドペナルティ | テクニカル(65点内) | 正のスコア（12点）に再設計 |

## 出力インターフェース

```typescript
interface ScoringGateResult {
  passed: boolean;
  failedGate: string | null; // "liquidity" | "spread" | "volatility" | "earnings" | "dividend"
}

interface NewLogicScore {
  totalScore: number;
  rank: "S" | "A" | "B" | "C" | "D";
  gate: ScoringGateResult;
  trendQuality: {
    total: number;        // 0-40
    maAlignment: number;  // 0-18
    weeklyTrend: number;  // 0-12
    trendContinuity: number; // 0-10
  };
  entryTiming: {
    total: number;           // 0-35
    pullbackDepth: number;   // 0-15
    breakout: number;        // 0-12
    candlestickSignal: number; // 0-8
  };
  riskQuality: {
    total: number;            // 0-25
    atrStability: number;     // 0-10
    rangeContraction: number; // 0-8
    volumeStability: number;  // 0-7
  };
}
```

ランクは現行の4段階（S/A/B/C）から5段階（S/A/B/C/D）に変更。`ScoringRecord`のrank列は`String`型のため、スキーマ変更不要。

| 指標 | 必要日数 | データ不足時 |
|------|---------|------------|
| SMA75（MA配列） | 75日 | SMA25のみで評価（最大14点） |
| 週足SMA13 | 65日 | 0点（週足トレンド未評価） |

| SMA25上の連続日数 | 50日+ | 利用可能な日数でカウント |

## 実装方針

1. 現行の `technical-scorer.ts` を新しいカテゴリ構造に書き換え
2. ゲート判定は既存の即死ルールロジックを流用し、流動性をゲートに移行
3. 週足データは日足OHLCVから週足を合成して算出
4. バックテストのon-the-flyモードで新旧スコアリングを比較可能にする
5. 閾値・重みはバックテスト結果を基にチューニング

## 成功基準

- バックテストでPF >= 1.2（現行1.07から改善）
- 勝率35%以上を維持しつつ平均利益率を向上
- 500銘柄×12ヶ月で200トレード以上の統計的有意性
