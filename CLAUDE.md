# Stock Buddy プロジェクト固有の設定

## プロダクトコンセプト

### サービス概要

**Stock Buddy** - 株式投資初心者向けのAIデータ分析ツール

**キャッチコピー**: 「データを読み解く力を、あなたに」

AIが複雑な市場データをわかりやすく整理・翻訳し、ユーザー自身が判断できる環境を提供。専門用語も解説付きで学びながら、投資の知識が自然と身につく。

> **重要**: 本サービスは投資助言を行いません。売買の判断はユーザー自身の責任で行ってください。AIはデータの分析・翻訳・教育に徹し、特定の銘柄の売買を推奨しません。

**デザインイメージ**:

- トーン: 親しみやすい、信頼感のある、落ち着いた
- カラー: 信頼・安定を表すブルー系ベース、成長・上昇を表すグリーンをアクセント
- イメージ: データを一緒に読み解くパートナー
- アイコン: 上昇チャート + 本（学習）を組み合わせたデザイン

### ターゲットユーザー

**投資初心者〜トレーダー志望 - 「学びながら成長したい」人**

- 投資に興味があり、これからトレードしていきたい
- 専門用語も含めて投資の知識を身につけたい
- 実践的なスキルを段階的に学びたい
- リスクを理解した上で自分で判断できるようになりたい

### コアバリュー

1. **寄り添う**
   - 「投資、始めたいけど怖い？大丈夫です。一緒にデータを読み解きましょう」
   - AIが専門用語をわかりやすく翻訳
   - 専門用語には必ず解説を添える

2. **学べる**
   - テクニカル分析（チャートパターン、RSI、MACDなど）を解説付きで提供
   - 「なぜこのパターンが出現しているのか」を理解できる
   - 使いながら投資の知識が自然と身につく

3. **シンプル**
   - 複雑なデータをスコアとランクで直感的に把握
   - プリセット条件で簡単にスクリーニング

4. **自立を支援**
   - 判断材料を整理して提示
   - 最終判断はユーザー自身が行う
   - 使い続けることで自分で判断できる力が身につく

### 設計思想

#### 基本原則: AIはデータ分析・翻訳者

**「AIは答えを出さない。判断材料を整理してユーザーに差し出す」**

- **AIの役割**: データの分析・翻訳・教育（テクニカル分析の解説、財務データの要約、ニュースの整理）
- **ルールの役割**: リスク要因の検出・アラート表示（赤字、高ボラティリティ、急落などの事実を警告）

AIは「データをわかりやすく整理する」ことに集中し、売買の判断はユーザーに委ねる。
これにより、投資助言業に該当するリスクを排除し、ユーザーの自立を支援する。

#### ⛔ 絶対にやってはいけないこと（投資助言に該当）

- 売買の推奨（buy/sell/hold/stay/avoid 等のラベル）
- 具体的な価格予想（「XXX円まで上がる」等）
- 売買タイミングの指示（「今が買い時」「押し目買い」等）
- AIによる判断の自動上書き（強制補正）
- 特定銘柄の購入・売却提案

#### 🛠️ Claude Codeの実装ガイドライン（投資助言回避）

**投資助言に該当する機能の実装を依頼された場合、そのまま実装せず代替案を提案してください。**

ユーザーが意図せず投資助言に該当する機能を要求することがあります。その場合は実装を拒否するのではなく、本サービスの設計思想に沿った代替案を提案してください。

| 依頼内容（⛔NG） | 代替案（✅OK） |
|---|---|
| 「買い/売りのシグナルを表示して」 | テクニカル指標の状態を表示（例: 「RSI 75 — 過熱圏」）。判断はユーザーに委ねる |
| 「おすすめ銘柄をレコメンドして」 | スクリーニング条件に合致した銘柄を一覧表示。「おすすめ」ではなく「条件一致」として提示 |
| 「この銘柄は買うべきか判定して」 | ファンダメンタル・テクニカルデータを整理して表示。ポジティブ/ネガティブ要因を並列に提示 |
| 「目標株価を予測して」 | アナリストのコンセンサス目標株価を引用（出典明記）。予測ではなく参考情報として表示 |
| 「ポートフォリオを自動リバランスして」 | 現在の配分比率と偏りを可視化。「リバランスの検討ポイント」としてデータを提示 |
| 「損切りラインを設定して」 | ボラティリティやATRなどのデータを表示。リスク管理の考え方を教育コンテンツとして提供 |

**対応フロー:**

1. 依頼内容が投資助言に該当するか判断
2. 該当する場合 → 「この機能は投資助言に該当する可能性があります」と説明
3. 代替案を提案（データ表示・教育・情報整理の形に変換）
4. ユーザーの承認を得てから実装

**判断基準:**
- 「〜すべき」「〜がおすすめ」→ ⛔ 助言に該当
- 「〜という状態です」「〜というデータがあります」→ ✅ 情報提供

#### ❌ やってはいけないこと

- 専門用語を解説なしで使う（用語には必ず説明を添える）
- 複雑な機能を追加する

#### ✅ やるべきこと

- 専門用語＋分かりやすい解説をセットで提供（例:「ダブルボトム（2回底を打って反転するパターン）」）
- テクニカル分析の結果を初心者が理解できる形で表示
- スコア（0-100）やランク（A-E）でデータを直感的に表現
- リスク要因は注意フラグとして事実を表示（判断はユーザーに委ねる）
- 迷わせない導線設計
- シンプルで分かりやすいUI
- 免責表示を常に表示（「投資判断はご自身の責任で行ってください」）

#### 初心者向けの定義

**初心者向け ≠ 難しい言葉や論理を避ける**

- ✅ 複雑な指標も入れつつ、わかりやすく提示する
- ✅ 専門用語を使い、その意味と評価を添える
- ✅ 使いながら自然と専門用語を覚えていける設計

**表示形式の例:**

```
シャープレシオ: 1.2
（リスクに対するリターンの効率。1以上で優秀）
→ あなたのポートフォリオは効率よくリターンを得ています
```

1. **指標名** - 専門用語そのまま
2. **解説** - その指標が何を意味するか
3. **評価** - 良い/普通/注意など
4. **アクション** - 次に何をすべきか

### 機能方針

#### オンボーディング

- 質問は最小限（予算・投資スタイル）
- プリセットスクリーニング条件から選択（高配当・割安・成長株等）

#### 制限設計

- ポートフォリオ: 最大100銘柄
- ウォッチリスト: 最大100銘柄
- 追跡銘柄: 最大10銘柄
- 制限値は `lib/constants.ts` で一元管理

### 銘柄管理の定数

**銘柄管理に関する制限値は `lib/constants.ts` で一元管理されています。**

#### 定数一覧

```typescript
import {
  MAX_PORTFOLIO_STOCKS,    // 100 - ポートフォリオ上限
  MAX_WATCHLIST_STOCKS,    // 100 - ウォッチリスト上限
  MAX_TRACKED_STOCKS,      // 10  - 追跡銘柄上限
} from "@/lib/constants"
```

#### 銘柄登録API

銘柄登録は `/api/user-stocks` で実装されています。

- **ポートフォリオ**: `POST /api/user-stocks` with `type: "portfolio"`
- **ウォッチリスト**: `POST /api/user-stocks` with `type: "watchlist"`
- **追跡銘柄**: `POST /api/tracked-stocks`

#### チェックリスト

制限値を変更する場合：
- [ ] `lib/constants.ts` の値を変更
- [ ] 関連するAPIやUIが正しく動作するか確認

### 仕様書の管理

**機能の追加・変更時は必ず `docs/specs/` の仕様書を更新してください。**

- **新機能追加時**: 該当する仕様書に機能を追記。新しい機能カテゴリの場合は仕様書ファイルを新規作成し、`main.md` の機能一覧にもリンクを追加
- **既存機能の変更時**: 該当する仕様書の記載内容を実装に合わせて更新
- **API追加・変更時**: エンドポイント、リクエスト/レスポンス仕様を更新
- **データモデル変更時**: 該当する仕様書のデータモデルセクションを更新
- **バッチ処理追加・変更時**: `batch-processing.md` のワークフロー一覧とデータフローを更新
